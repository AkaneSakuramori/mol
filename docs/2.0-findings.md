# 2.0 Real-World Findings Log

A record of genuine limitations discovered by building real software in Ulang, and the
improvements made in response. This log is a defining outcome of the 2.0 milestone: every
entry is a shortcoming that only surfaced through practical use, not speculation.

Format: each entry names the project that exposed it, the problem, and the fix.

---

### F1 — `list.sort` ignored its comparator
- **Exposed by:** `wordstats` (sorting `(word, count)` pairs by frequency).
- **Problem:** `list.sort()` was implemented as a bare `sorted(obj)`, ignoring any argument
  and throwing `TypeError` when the elements were structs (no natural ordering).
- **Fix:** `list.sort(cmp)` now accepts an optional comparator `fn(a, b) -> int` (negative /
  zero / positive) or a boolean, in addition to the no-argument natural sort. Implemented
  once in `src/builtins_mod.py` (`_sort`), shared by the interpreter and the VM.
  Backward-compatible. (CHANGELOG: "toward 2.0".)

### F2 — match arms don't take brace blocks (ergonomics, documented, no change)
- **Exposed by:** `calc` (wanting a multi-statement `match` arm).
- **Finding:** match arms are a single expression or an indented block (per the grammar);
  the `{ ... }` brace form is only for lambda bodies. This is a deliberate grammar choice,
  not a bug. Resolved by writing idiomatic arms; recorded so the constraint is explicit.

### F3 — missing common string methods (`substring`, `index_of`, `repeat`)
- **Exposed by:** `calc`, `rpn`, `jsonfmt` all hand-rolled substring/number-formatting
  helpers; `table` needed column padding.
- **Problem:** the string type had `split`/`trim`/`replace`/`contains`/`starts_with`/
  `ends_with`/`chars`/`upper`/`lower` but no way to take a slice, find a substring index, or
  repeat a string — forcing every text-processing program to reimplement them by hand.
- **Fix:** added `str.substring(start[, end])` (bounds-clamped), `str.index_of(sub)`
  (returns `-1` when absent), and `str.repeat(n)`, in `src/builtins_mod.py`, shared by the
  interpreter and VM. Backward-compatible. `table` uses them directly for padding.
